## TL;DR

Walker emits `Category::Index kind="enrichment"` envelopes when a vault's `repo:` frontmatter field matches the current project root. Frontend `EnrichmentStore` joins envelopes onto `Tree.svelte` node paths via string-equal `fs_path` lookup. Enriched fs nodes render a **single uniform muted-amber dot** (no glyph, no per-vault-kind color) right of the node label, with **dual tooltip** (SVG `<title>` for screen readers + SVG `<foreignObject>` for styled visual on hover) listing all `{vault_id, vault_kind, tags}` for that path.

Path-key contract: walker emits `fs_path` = canonicalized absolute project-root path (forward-slash-normalized via `dunce::canonicalize` + replace + trim). Tree's root `node.path` carries the same shape (also via `dunce::canonicalize` at lib.rs:923). Join is string equality.

Shipped in commits `96b0de2` (8.6.1 store + subscription) → `331412e` (8.6.2 render) → `e935fe2` (8.6.3 walker publish + lib.rs wiring, TRANCHE) → THIS (8.6.4 spec patch + IndexEnrichmentPayload doc fix).

## The questions Phase 8.6 needed to answer (Q1–Q6)

The 8.6 plan went through 2 rounds of opus Critic adversarial review (`/aegis --plan --upgrade`). Critic v1 caught a mismatch with the walker's actual location + a P0 bug where the walker silently no-op'd on production vaults (`abce02c` 8.5b telegraphic-parser fix). Critic v2 caught: (C1) my proposed glyph-in-dot was geometrically impossible (6px glyph in 4.5px dot); (C2) my proposed `fs_path=""` sentinel for project-root would not match Tree's canonical-absolute-path root node. Plan v3 locked the simpler answers below.

### Q1 — Visual treatment per `vault_kind`

**Locked:** Single uniform muted-amber `#5a4410` dot. **No glyph.** Vault kind disclosed via tooltip text only.

- Hex `#5a4410` is §10.1 "meta / timestamps" lane — semantically right (enrichment IS metadata about the fs node).
- Avoids §10.1 lane-color collisions (red→error, amber→Aegis, etc.) that earlier glyph palettes risked.
- Geometry: dot radius `FILE_R / 2` = 2.25px → 4.5px diameter. Glyph attempt at 6px would have overflowed the dot entirely (Critic v2 C1 caught this math).
- WCAG 1.4.1: dot's PRESENCE (not its color) is the binary signal. Color + ARIA + tooltip text together cover the signaling. Color alone is not load-bearing.
- ARIA label on the outer dot `<g>`: `aria-label="Enriched"`.

### Q2 — Multiple enrichments per node

**Locked:** Single dot per node regardless of count. Tooltip discloses multiplicity.

- Stacking + overflow caret was considered; rejected for v1 because the dot is already small (4.5px) and label-area width is bounded at SVG_WIDTH=320px.
- Future v1.x: stacking with max-N + overflow `…` caret if user signals desire.

### Q3 — Path match strategy

**Locked:** Exact path only.

- Prefix-match enrichment ("vault enriches `crates/rift-bus/` → all children inherit") was considered; deferred to v1.x via DEFERRED.md (lazy — open the entry only if user asks).
- Exact-only avoids visual noise + ambiguity ("did this dot mean THIS file or its parent dir?").

### Q4 — Click / drag behavior

**Locked:** Hover-tooltip only. Drag = existing fs-path drag from Phase 6.6.

- Click-to-open-vault from the dot was considered; deferred to Phase 8.7 (alongside drag-into-terminal vault-payload work).
- Dot SVG carries `pointer-events: none` so drag continues to bubble to the parent `<g class="tree-node">` ondragstart handler. Without this, dragging from a dot would silently break drag (Critic v2 L3).

### Q5 — Enrichment source

**Locked:** Existing vault-walker at `crates/rift-bus/src/translators/vault_walker.rs` (PUBLIC bus crate, NOT `crates/rift-aegis/`).

- Walker consumes the already-shipped `VaultMeta.repo: Option<String>` field populated by the 8.5b telegraphic-frontmatter parser (`abce02c`). No new parsing.
- Walker spawn signature gains a third parameter: `project_root: Option<PathBuf>`. `None` → no enrichment emit (test fixtures and bare installs); `Some(p)` → per-vault `repo:` comparison.
- Future enrichment sources (CLAUDE.md scanner, `.abyssal/` sidecar files, etc.) are additive — they wire to the same `publish_index_enrichment` API. Out of scope for v1.

### Q6 — Path-key contract (NEW in v3, resolves Critic v2 C2)

**Locked:** Walker emits `fs_path` = canonical absolute project-root path, forward-slash-normalized.

- Walker normalizes via `dunce::canonicalize` (NOT `std::path::absolute` — Critic v2 M2; `dunce` is already a workspace dep at lib.rs:790, 923 and handles UNC + long-path Windows correctness).
- Normalize step (applied symmetrically to BOTH the canonicalized `meta.repo` and the canonicalized `project_root`): `replace('\\', "/")` + `trim_end_matches('/')`. String comparison is then exact.
- Tree.svelte's root `node.path` is itself canonicalized via the same `dunce::canonicalize` chain at lib.rs:923. Identical normalization on both sides → string-equal join holds.
- Sub-path enrichment via vault `source:` field deferred to v1.x.

## Wire contract — `Category::Index kind="enrichment"`

```json
{
  "fs_path":    "C:/Users/Critek/Documents/Abyssal_Arts_main/Projects/Rift_TerminalV2",
  "vault_id":   "p006",
  "vault_kind": "project",
  "tags":       []
}
```

| Field | Type | Notes |
|---|---|---|
| `fs_path` | string | Canonical absolute project-root path, forward-slash-normalized (no trailing slash). String-equal join with `Tree.svelte`'s root `node.path`. |
| `vault_id` | string | Vault identifier from frontmatter (e.g. `"p006"`, `"pr003"`). On vault deletion, walker emits this real id via path→id cache (Critic v2 M5). |
| `vault_kind` | string | Long form derived from id prefix by walker's `derive_vault_type`. One of `"project"`, `"practices"`, `"research"`, `"skill"`, `"lore"`, `"agent"`, `"hook"`. |
| `tags` | string[] | Vault-sourced tags. Empty in v1; future fields can populate. |

`bus.ts` deserializer does NO key transformation — payloads pass through as raw JSON the Rust side built. Frontend `EnrichmentEntry` interface is snake_case throughout (Critic v2 H2 verification).

## Critic-rounds appendix — what each pass caught

### Round 1 (`/aegis --plan` v1 → opus Critic)

- **CRITICAL #1 — wrong walker location.** v1 plan said `crates/rift-aegis/src/<vault_walker>.rs`; reality `crates/rift-bus/src/translators/vault_walker.rs`. Verified inline; resolved by direct read.
- **CRITICAL #2 — walker silently broken on production vaults.** Walker required `---`-fenced YAML frontmatter; production vaults use telegraphic `VAULT: <id> | …` line-1 header. 0/41 vaults parsed. Fixed by `abce02c` (Phase 8.5b telegraphic parser, dual-format dispatcher).

### Round 2 (`/aegis --plan` v2 → opus Critic)

- **CRITICAL #1 — glyph geometry impossible.** v2 specified `FILE_R/2`=2.25px dot with 6px glyph inside. Resolved by dropping glyph entirely (Q1 v3 = uniform amber, no glyph).
- **CRITICAL #2 — `fs_path=""` sentinel won't join Tree's root.** Tree's root path is canonical-absolute via `dunce::canonicalize`. Sentinel matches nothing. Resolved by Q6 — emit canonical-absolute path itself.
- **HIGH H1** — `spawn_vault_walker` signature change requires lib.rs:982-985 edit (2-file scope, not 1-file). Spec'd in 8.6.3.
- **HIGH H2** — TS interface MUST be snake_case (verified `bus.ts` does no key transformation). Spec'd in 8.6.1.
- **HIGH H3** — 8.6.3 tests must use telegraphic fixtures (not YAML) so they exercise production parse path. Spec'd in 8.6.3.
- **HIGH H4** — tooltip lifecycle ambiguous. Resolved by SVG `<foreignObject>` (auto-moves with SVG transform; no `getBoundingClientRect()` overlay drift on scroll) + SVG `<title>` for a11y.
- **MEDIUM M1** — Svelte 5 reactivity requires Map assign-replace. Spec'd inline (`store.map = new Map(this.map).set(...)`).
- **MEDIUM M2** — use `dunce::canonicalize` (workspace dep, lib.rs:790,923 consistency) NOT `std::path::absolute` (semantics + version-pin risk). Spec'd in Q6.
- **MEDIUM M3** — 8.6.2 needs explicit test surface. Pivoted to `enrichmentUtils.ts` extracted helpers because `@testing-library/svelte` not installed.
- **MEDIUM M4** — once-per-session diagnostic logs via AtomicBool `compare_exchange`. Spec'd in 8.6.3.
- **MEDIUM M5** — walker's vault.update/deleted emits `path.file_stem()` fallback id. Path→id cache fix so emit is real `meta.id`. Spec'd in 8.6.3.
- **LOW L1** — `IndexEnrichmentPayload.fs_path` doc said "project-relative" but plan emits absolute. Fixed in 8.6.4 (this commit).
- **LOW L2** — walker's `vault_kind` is long form ("project") not prefix ("p"). Frontend interface uses long form. Doc updated in 8.6.4.
- **LOW L3** — `pointer-events: none` on dot SVG so drag bubbles to parent. Spec'd in Q4.
